# 📊 Аналітика ефективності ланцюжка постачання та роздрібних продажів

Проєкт наскрізної (End-to-End) аналітики даних, присвячений дослідженню комерційного розподілу продажів, тенденціям сезонності та оптимізації зворотної логістики (аналіз обороту та повернення тари). Проєкт охоплює повний цикл: від первинної обробки даних та виявлення аномалій у **Python (Jupyter Notebook)** до побудови інтерактивного дашборду для керівництва в **Tableau**.

## 🚀 Огляд проєкту та ключові інсайти

У цьому проєкті проаналізовано дані про дистрибуцію та логістику напоїв за період 2017–2020 рр., що дозволило виявити чіткий структурний поділ у бізнес-моделі компанії: **Пряма комерційна логістика** проти **Зворотної логістики (управління тарою)**.

### 💡 Ключові знахідки (Insights):
* **Літній пік попиту:** Операційна діяльність компанії має яскраво виражену сезонність із потужним стрибком обсягів кожного **липня** (переважно за рахунок категорій *Beer* та *Non-Alcoholic*). Логістичні та складські потужності мають масштабуватися динамічно саме під цей період.
*  ![](https://github.com/IvanRadelytskyi/Supply-Chain-Returns-Performance-Dashboard/blob/main/%D0%9F%D0%BE%D1%81%D1%82%D0%B0%D1%87%D0%B0%D0%BD%D0%BD%D1%8F.png)
* **Аномалія категорії "Dunnage":** Первинні сирі дані показували аномально високий загальний коефіцієнт повернень, що викривляло реальну картину бізнесу. Завдяки ізольованому аналізу було визначено, що категорія `DUNNAGE` відображає **зворотну логістику** (рух порожніх кег/тари назад постачальникам, переважно *Anheuser Busch Inc*). Система коректно фіксує це як від'ємний об'єм (зворотний потік), а не як фінансові збитки чи брак продукції.
* ![](https://github.com/IvanRadelytskyi/Supply-Chain-Returns-Performance-Dashboard/blob/main/%D0%9E%D0%B1%D0%BE%D1%80%D0%BE%D1%82.png)
* **Домінування B2B транзиту:** Категорія `KEGS` реалізується виключно через складські відвантаження (`Warehouse Sales`), тоді як рух у роздрібній мережі майже нульовий. Це доводить чисту B2B модель виконання гуртових контрактів.
* **Драйвери ринку:** Пиво (`Beer`) виступає головним драйвером фізичних об'ємів (бренди *Corona Extra* та *Heineken* міцно утримують лідерство), тоді як міцний алкоголь (`Liquor`) демонструє високу централізацію навколо одного монополіста (*Tito's Handmade*).
*  ![](https://github.com/IvanRadelytskyi/Supply-Chain-Returns-Performance-Dashboard/blob/main/%D0%A2%D0%BE%D0%BF.png)

## 🛠️ Технологічний стек та робочий процес

### 1. Конвеєр даних та розвідувальний аналіз (EDA) — `Python`
Виконано в Jupyter Notebook для аудиту якості даних перед початком візуалізації:
* **Очищення та парсинг даних:** Оброблено пропущені значення (валідовано за допомогою теплової карти `seaborn.heatmap`) та об'єднано окремі атрибути `YEAR` і `MONTH` у єдиний часовий формат (datetime).
* ![](https://github.com/IvanRadelytskyi/Supply-Chain-Returns-Performance-Dashboard/blob/main/%D0%9F%D0%BE%D1%80%D0%BE%D0%B6%D0%BD%D1%96.png)
* **Пошук аномалій:** Ізольовано від'ємні транзакції для визначення першопричини зворотних потоків та відстеження обігу тари.
* **Аналіз часових рядів та портфеля брендів:** Сегментовано Топ-5 постачальників та побудовано графіки динаміки обсягів їх дистрибуції на 4-річному таймлайні.

### 2. Бізнес-аналітика та архітектура дашборду — `Tableau`
Спроектовано звіт для топ-менеджменту з фокусом на великі візуальні метрики (BANs — Big Angry Numbers) та можливості глибокої фільтрації:
* **Складні обчислення:** Впроваджено вирази **`FIXED LOD`** для збереження глобальних коефіцієнтів повернення незалежно від локальної фільтрації вимірів на дашборді.
* **Динамічне ранжування:** Використано аналітичні віконні функції **`RANK()`** для динамічного перерахунку картки **Top 3 Items** на основі вибраної користувачем категорії товарів.
